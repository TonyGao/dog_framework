### 2️⃣ 评分模型（示例：加权线性回归）

\[
\text{综合分数}_i = 
\sum_{j=1}^{8} w_j \times Z_{ij}
\]

- \(Z_{ij}\) 为维度 \(j\) 的标准化得分（0‑100）。  
- 通过历史数据做线性回归校准，使模型对未来绩效预测误差最小。  

#### 2.1 数据预处理

| 步骤 | 说明 |
|------|------|
| **归一化** | 对每个子指标进行 Z‑score 或 Min‑Max 标准化，避免量纲差异影响综合分数。 |
| **缺失值填补** | 使用历史趋势插补或使用平均值替代（在敏感维度如心理状态建议使用最近一周的平均）。 |
| **异常检测** | 通过 Z‑score > 3 或基于业务规则（如“超出 KPI 天花板”）剔除/标记异常数据。 |

#### 2.2 机器学习增强（可选）

- **特征工程**：将过往年度的每项子指标、岗位变化、工作时长等作为特征，训练随机森林或梯度提升回归模型预测当前员工的“未来6个月绩效风险”。  
- **情感分析**：对每日情绪自报文字进行 LSTM/BERT 情感评分，再转为量化情绪指数。

| 环节 | 具体实现手段 |

| **4. 计算引擎** | Python (pandas, scikit‑learn) 做特征加工；训练模型后导出 ONNX / TensorFlow SavedModel，部署在 Kubernetes 上作为服务接口。 |

| **6. 隐私/合规** | - 数据最小化原则：仅收集评估必需的字段<br>- 加密传输（TLS）和存储（HSM）<br>- 实现“可解释”模型：每张综合分数输出权重来源明细。<br>确保符合 GDPR、MACIA（香港）等相关法规。 |
| **7. 闭环反馈** | 每月推送“评估报告”，并嵌入调查表（NPS 对评估透明度），AI 根据低分员工的情绪指数提醒经理进行辅导。 |

## 四、示例：某位工程师的 6 月综合分数计算流程

1️⃣ **数据抓取**  
- 项目交付量：90% KPI（基准 80%） → Z=95  
- 质量缺陷率：3.2/千名（基准 5）→ Z=85  
- 客户满意度（NPS）：+30（基准 +20）→ Z=87  

2️⃣ **主观评估**  
- 情绪日均指数（每日1‑5星平均）：4.2 → Z=90（优秀）  
- 团队信任评分（360°）：4.5/5 → Z=92  
- 提案采纳率：78% → Z=88  

3️⃣ **维度加权（示例）**  
| 维度 | 权重 | Z | 贡献 |
|------|------|---|-------|
| A. 长期能力 | 18% | 90 | 16.2 |
| B. 工作水平 | 20% | (95+85+87)/3 ≈ 89 → Z≈ 84 | 16.8 |
| C. 心理状态 | 10% | 90 | 9.0 |
| D. 团队作用 | 8% | (协作次数、贡献占比) Z≈85 → 6.8 |
| E. 影响力 | 8% | 88 → 7.0 |
| F. 亲和力 | 8% | 92 → 7.4 |
| G. 职业发展潜力 | 8% | 75 → 6.0 |
| H. 工作态度/价值观 | 6% | 80 → 4.8 |

**总得分** = 16.2+16.8+9.0+6.8+7.0+7.4+6.0+4.8 ≈ **83.0（满分100）**  

> 该员工综合分数 83，说明在绩效与心理上表现优秀，仍有提升空间在于职业发展潜力和价值观认同等。
